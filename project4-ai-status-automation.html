Project Overview

During a healthcare provider's 18-month IT infrastructure modernization program across 15 hospitals and 50+ stakeholders, I identified a critical bottleneck: manual weekly status report compilation was consuming 4-5 hours every week.

Rather than hire additional program coordinators, I built a Gen AI-powered automation solution using Claude API to auto-generate comprehensive status reports from real-time program data sources. The solution integrated with Jira, deployment logs, vendor updates, and risk assessments to produce executive-quality reports in minutes.

This project demonstrates how strategic application of Gen AI can accelerate program delivery and free PMs to focus on strategic decisions rather than administrative tasks.

The Challenge

  • Manual Labor Bottleneck: Weekly status reports required manually aggregating data from 5+ sources (Jira, Slack, vendor emails, deployment logs, risk registers)
  • Consistency Issues: Reports varied in format, depth, and tone across 15 hospital sites — stakeholders wanted standardized, professional output
  • Time Constraint: 4-5 hours per week spent on compilation that could be spent on strategic problem-solving and stakeholder engagement
  • Scale Challenge: As program expanded, manual reporting became increasingly unsustainable without hiring additional staff
  • Accuracy Risk: Manual data entry and aggregation introduced typos, missed updates, and inconsistent metrics

My Approach & Solution

Problem Analysis: I identified that status reporting followed a predictable pattern: extract data → synthesize narrative → format output. This was ideal for AI automation — the pattern was repetitive, the data sources were structured, and the output requirements were clear.

Architecture Design: Built a Python integration that:

  • Extracted weekly data from Jira (milestone progress, ticket velocity)
  • Pulled deployment logs and success metrics from CI/CD systems
  • Aggregated vendor update summaries from email and Slack
  • Retrieved risk register and issue tracker entries
  • Fed all data to Claude API with a structured prompt
  • Generated polished status reports in 5-10 minutes

Claude API Implementation: Used Claude Opus for deep reasoning and synthesis. The system prompt enforced executive-level language, consistent structure, and contextual insights:

import anthropic def generate_status_report(program_data): """ Generate weekly status report using Claude API program_data: dict containing Jira tickets, deployments, risks, vendor updates """ client = anthropic.Anthropic(api_key="your-api-key") # Prepare structured data for Claude prompt = f""" You are an expert healthcare IT program manager. Generate a professional weekly status report using the following data. Include executive summary, milestone progress, risks, blockers, and next steps. Program Data: - Milestone Progress: {program_data['milestones']} - Deployments This Week: {program_data['deployments']} - Active Risks: {program_data['risks']} - Vendor Updates: {program_data['vendor_updates']} - Team Velocity: {program_data['velocity']} - Blockers: {program_data['blockers']} Generate a concise, professional report suitable for C-level stakeholders across 15 hospital sites. Include specific metrics and data points. Format with clear sections: Executive Summary, Milestone Status, Risk Summary, This Week's Wins, Blockers & Mitigation, Next Week's Priorities. """ message = client.messages.create( model="claude-opus-4-20250805", max_tokens=1500, messages=[ { "role": "user", "content": prompt } ] ) return message.content[0].text # Usage program_data = { "milestones": "Hospital 1-5 network upgrade 80% complete, Hospital 6-10 in design phase", "deployments": "3 hospitals went live this week, 0 incidents", "risks": "Supply chain delay on network switches (Yellow), 2 vendors behind schedule (Yellow)", "vendor_updates": "Cisco on track, Fortinet delayed 1 week, Network team adds 2 engineers", "velocity": "92% on-time delivery rate across all sites", "blockers": "Awaiting hospital IT director sign-off on Hospital 8 cutover plan" } report = generate_status_report(program_data) print(report)

Human Review Process: While AI generated the initial draft, I maintained human oversight. A PM review step ensured accuracy, added contextual insights, and customized tone for different stakeholder groups.

Integration & Workflow: The system ran every Friday afternoon, generating reports by 4 PM for Monday stakeholder meetings. Reports were auto-formatted and distributed to 50+ stakeholders via email.

Key Outcomes:

  • Reduced manual report compilation from 4.5 hours to 30 minutes per week — 80% time savings
  • Improved consistency across all 15 hospital sites — standardized format, tone, and depth
  • Achieved 95% stakeholder satisfaction with report quality and timeliness (measured via survey)
  • Freed up 190+ hours annually for strategic PM work (vendor negotiations, risk mitigation, stakeholder alignment)
  • Demonstrated cost savings equivalent to 1 FTE coordinator (~$120K annually)
  • Established replicable AI automation framework adopted for other repetitive PM tasks
  • Positioned healthcare provider as innovative in Gen AI adoption among industry peers

Key Learnings

  • AI Amplifies, Not Replaces: Claude API generated excellent first drafts, but human judgment remained critical for nuance, stakeholder context, and strategic insights. The best results came from AI-human collaboration.
  • Data Quality Matters: The quality of generated reports directly reflected the quality of input data. Clean, structured data from Jira and logs meant better output. Garbage in, garbage out.
  • Prompt Engineering is PM Skill: Writing clear, structured prompts for Claude was remarkably similar to writing clear project briefs — specificity, context, and expected output format all matter.
  • Change Management Required: Stakeholders initially skeptical of "AI-generated" reports. Building trust required transparency about how reports were created and showing that human oversight remained.
  • Identify Bottlenecks Early: The best candidates for AI automation are repetitive, high-volume, standardized tasks. Reporting fit perfectly; strategic decisions don't.
  • Scale Automation Strategically: After proving the model with status reports, we applied the same Claude API framework to vendor performance summaries, risk assessments, and stakeholder communication templates.
